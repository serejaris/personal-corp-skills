# To PRD

Собирает из уже обсуждённого в сессии (и из кодовой базы) один PRD в папке проекта — без нового интервью, только синтез. Агент смотрит репозиторий, согласует с тобой швы для тестирования и пишет `PRD.md` в корне рабочей папки фичи или демо.

**Когда звать:** после [grill-me](../grill-me/) или любого длинного разговора о фиче, когда решения уже есть, но нет единого документа для команды и следующего шага.

**На выходе:** `PRD.md` — problem/solution, длинный список user stories, implementation и testing decisions, out of scope.

**Дальше по цепочке:** [to-issues](../to-issues/) режет PRD на вертикальные срезы в `tasks/NN-slug.md`.
